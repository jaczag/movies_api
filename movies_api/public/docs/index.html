Introduction

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Base URL

http://localhost

Authenticating requests

This API is not authenticated.

Endpoints

POST api/v1/auth/register

Example request:
curl --request POST \
    "http://localhost/api/v1/auth/register" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"first_name\": \"rdppnnqh\",
    \"last_name\": \"xbupyarhgscmdelxukranurypoglbdcvgdpvkwzoejuelggu\",
    \"email\": \"grtqbfyvcgowalewwgthaburjhfblssscgcw\",
    \"password\": \"fxppdftzutiedyd\",
    \"password_confirmation\": \"atque\"
}"

const url = new URL(
    "http://localhost/api/v1/auth/register"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "first_name": "rdppnnqh",
    "last_name": "xbupyarhgscmdelxukranurypoglbdcvgdpvkwzoejuelggu",
    "email": "grtqbfyvcgowalewwgthaburjhfblssscgcw",
    "password": "fxppdftzutiedyd",
    "password_confirmation": "atque"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Request      

POST api/v1/auth/register

Body Parameters

first_name   string   

Must contain only letters. Must not be greater than 20 characters.

last_name   string   

Must not be greater than 50 characters.

email   string   

Must be a valid email address. Must not be greater than 50 characters.

password   string   

Must be at least 7 characters. Must not be greater than 20 characters.

password_confirmation   string   

The value and password must match.
